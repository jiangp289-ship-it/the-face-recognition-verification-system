# 人脸鉴伪系统后台高保真原型

## 运行方式

1. 直接打开 `prototype/index.html`。
2. 或在 `prototype/` 目录启动任意静态文件服务后访问 `index.html`。

## 页面范围

- 业务请求概览：`#/dashboard`
- 安全态势概览：`#/faceDeepfakeOverview`
- 产品管理：`#/product`
- 业务管理：`#/business`
- 策略配置：`#/financialLivenessRisk`
- 人脸黑名单库：`#/risk-list`
- 操作日志：`#/operationrecord`
- 系统管理：`#/system`

## 功能说明

- 页面按本轮二次优化后的 `docs/UX_SPEC.md` 与 `docs/DESIGN.md` 第 24.15 节实现，视觉仅使用设计 token 和统一组件。
- 业务请求概览支持时间类型「日 / 月」、业务列表选择、检测量 / 通过量图例和趋势联动。
- 安全态势概览支持「请求趋势 / 拦截趋势」TAB，TAB 只切换趋势模块；安全态势详情常驻展示设备风险详情和人脸静默风险详情。
- 产品管理支持查询、创建、编辑、查看业务跳转和操作日志跳转。
- 业务管理已删除状态筛选框和业务列表状态列。
- 策略配置支持查询、重置、创建、编辑、启停、删除，创建/编辑弹窗按基础信息、作用范围、风控条件、处置动作分组。
- 人脸信息库仅包含人脸黑名单库。
- 所有业务数据均为虚构演示数据，不包含真实客户信息。

## 已覆盖交互

- 页面导航、一级/二级菜单展开收起、当前菜单高亮、面包屑和页面标题。
- 查询、筛选、tabs、概览图表 hover 提示、分页、导出 toast。
- 产品、业务、策略、名单、账号的创建和编辑 mock。
- 策略配置作业类型联动：全局禁用产品编号 / 业务ID；产品或业务可填写；编辑态规则名称和产品编号 / 业务ID 不可编辑。
- 批量操作菜单、删除确认、启停确认、关闭确认。
- 表单校验、loading、empty、error、toast、modal、drawer、基础响应式。

## 已知限制

- 原型为纯前端 mock，无真实后端接口。
- 导出、上传、删除、停用、关闭、批量配置均只作用于浏览器内存数据。
- 筛选、保存和状态切换刷新页面后不会持久化。
- 图表为 HTML/CSS 模拟展示，不接入真实图表库或真实数据源。
- 字段矩阵已依据主控放行后的源字段矩阵与 UX 规格校准；业务管理 K-009 使用 5 套独立 schema。
- 源字段产品命名在原型 UI 中统一为“人脸鉴伪”，配置区 help 文案保留该映射说明。
- 业务列表操作列折叠为“更多”，其中动作配置、接口定制、人脸库、试用/正式配置等源站下钻入口映射到配置抽屉内对应分组。
- 人脸检索的人脸库图片预览补充 `operation.closePreview` / “人脸预览关闭”，点击“关闭预览”仅关闭预览态。
